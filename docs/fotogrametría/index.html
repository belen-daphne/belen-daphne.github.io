2.1: Fotogrametría

Comenzamos el semestre descubriendo una nueva herramienta. La fotogrametría consiste de un proceso en el cuál un programa transforma fotos tomadas de un objeto desde muchos ángulos para crear un objeto en 3d en forma de malla con información sobre los colores. La cantidad de fotos requeridas para llevar a cabo el proceso puede variar según el programa, la complejidad del objeto y la definición que queramos en nuestro producto final. En el programa ReCap Photo funcionan desde 20 fotos, y con la licencia de estudiante se pueden llegar hasta las 100. Probando este herramienta, genere dos objeto. El primero de estos fue un plato de verduras: con lechuga escarola, tomate y zanahorias.

Siendo este el mejor ángulo para mostrarlo, debo mostrar las fallas también.

Con esta iteración descubrí que los platos hondos eran mala idea y que debía tener una distribución más pareja de los ángulo de las fotos. Estoy contenta con el resultado gracias a la textura que capturo de la lechuga. Este intento fue compuesto por 96 fotos.

Otro intento fue una prueba para comprobar el cambio que se producía cambiando la cantidad de fotos. Este intento creo que está compuesto por 27 fotos, más o menos. Es un sartén con huevo, tomate y pan (que se me quemó). Varía un montón aunque se pueden llegar a un modelo bastante decente.

Por último, buscando las imagenes para el post descubrí una imagen que estaba guardada en la carpeta de ADrive. Me parecio sumamente interesante como captaba diversas partes del objeto.

Me preguntó si serán las partes que se repiten dentro de las fotos, con las cuales el programa se guía. O si será un mapeo de texturas. No lo sé.
